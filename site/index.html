Infra for parallel AI coding agents

Run dozens of Claude Code sessions unattended via tmux. Self-healing watchdog, web dashboard, mobile PWA, agent-to-agent orchestration.

  • Self-healing watchdog -- auto-compacts, restarts, and replays on crash
  • Mobile-first PWA with offline queue and Background Sync
  • Single Python file, zero dependencies beyond python3 and tmux

How it works
Everything under the hood
A complete control plane for AI coding agents.
Self-healing background watchdog -- context low -> /compact; thinking-block corruption -> restart + replay; stuck waiting -> auto-respond. All silent.
Status from terminal output, not an API -- parses ANSI-stripped tmux output to detect working / waiting / idle. No hooks, no patches, no modifications to Claude Code.
YOLO auto-responder -- detects blocking tool-approval prompts via the "Esc to cancel" marker (never appears on open-ended questions) and auto-answers. Truly headless.
Agent-to-agent orchestration -- $AMUX_SESSION and $AMUX_URL injected at startup; a shared global memory file gives every agent the full REST API reference. Agents discover peers and delegate work without being told how.
Atomic task claiming -- no lock broker -- POST /api/board/:id/claim is a SQLite CAS. Dozens of agents race for the same queue -- no Redis, no duplicated work.
Single Python file, zero build step -- Python server + full dashboard as inline HTML/CSS/JS. Edit it; it restarts via os.execv on save. Only deps: python3 and tmux.
Parallel agents, permanent conversation IDs -- any number of sessions, each with a UUID that survives stop/start. amux exec myproject -- "..." registers, starts, and sends in one shot.
Conversation fork -- clone any session's JSONL history into a new session on a separate branch. Falls back to scrollback when JSONL isn't available.
Multi-pane workspace -- full-screen tiled layout, each pane with its own send bar. Drag to rearrange; layout and named profiles persist.
Live peek -- including stopped sessions -- scroll full scrollback, search, preview files, fire quick-actions without attaching. Output snapshots to disk every 60s and survives restarts.
Token tracking from JSONL -- daily spend per session with cache reads broken out. Deduplicates by signature so restarts don't double-count.
People tab -- personal CRM -- contacts with health indicators (days since last touch), full interaction log, follow-up queue, and auto-assigned PPL-N IDs. Fully accessible via CLI: amux crm list/add/log/fu. Optimistic UI throughout.
Kanban board with iCal sync -- SQLite-backed with auto-generated keys, per-agent ownership, and custom columns. Exports iCal to S3 -- due dates appear in Google/Apple Calendar.
Built-in cron -- no crontab, no systemd -- schedule one-time or recurring commands in any session. Next-run computed atomically in SQLite with missed-fire recovery, delivered via tmux.
Git-aware conflict detection -- warns when two sessions share the same dir + branch, with a one-click helper to isolate each agent onto its own branch.
Web dashboard + PWA -- installable on iOS and Android. Background Sync replays queued commands on reconnect. TLS auto-provisioned: Tailscale cert -> mkcert -> self-signed fallback.

One command, any action

orchestration example
# Send a task to another session
$ curl -sk -X POST \
-H 'Content-Type: application/json' \
-d '{"text":"implement login"}' \
$AMUX_URL/api/sessions/worker-1/send

# Atomically claim a board item
$ curl -sk -X POST \
$AMUX_URL/api/board/PROJ-5/claim

# Watch another session's output
$ curl -sk "$AMUX_URL/api/sessions/worker-1/peek?lines=50"
