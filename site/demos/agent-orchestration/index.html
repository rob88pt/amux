Agent-to-Agent Orchestration

Agents discover peers and delegate work via REST API. Every session gets $AMUX_SESSION and $AMUX_URL at startup plus the full API reference in global memory.

How it works

  1. Every agent starts with $AMUX_SESSION (its own name) and $AMUX_URL injected
  2. Global memory gives every agent the full REST API reference automatically
  3. Orchestrators delegate: POST /api/sessions/worker/send with plain-English instructions
  4. Workers claim board tasks, report back via POST /api/sessions/orchestrator/send

When to use this

  • Building a fan-out system: one orchestrator, many specialized workers
  • Running parallel feature branches with agents that report to a coordinator
  • Any task too large for a single context window

Example

# Orchestrator delegates to a worker
curl -sk -X POST $AMUX_URL/api/sessions/worker-1/send \
  -H 'Content-Type: application/json' \
  -d '{"text":"implement the auth endpoints in /src/routes/auth.ts and report back"}'

# Worker peeks at orchestrator output
curl -sk "$AMUX_URL/api/sessions/orchestrator/peek?lines=20"
